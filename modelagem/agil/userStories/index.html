    Histórias de Usuário

    1. Introdução

    As Histórias de Usuário (User Stories) são estratégias usadas para descrever de maneira breve, específica e detalhada as vontades do usuário (SERRANO; SERRANO, 2025). Elas são escritas partindo do ponto de vista do usuário sobre a aplicação final e sobre o que ele desejaria obter no sistema. São amplamente utilizadas em metodologias ágeis de desenvolvimento.
    Normalmente as Histórias de Usuário seguem o padrão:
    Eu, como <tipo de usuário>,
    quero <realizar uma ação>
    para <atingir um objetivo>.

    2. Metodologia

    As histórias de usuário foram confeccionadas por todos os membros do grupo de forma assíncrona, tendo como referência os requisitos elicitados em entregas anteriores do projeto.
    Para priorizar as histórias de usuário foi utilizada a técnica MoSCoW, que consiste em diferentes níveis de prioridade, com: deve ter (Must have), deveria ter (Should have), poderia ter (Could have) e não deveria (Won't have).
    A validação e a priorização das histórias foram realizadas com o cliente Gabriel Bevilaqua de forma presencial. A gravação do encontro pode ser observada abaixo.

    Tabela 42: US40 – Permitir compartilhamento do aplicativo com outras pessoas

    ID História de Usuário Critérios de aceitação Rastreabilidade Prioridade

    3.41 US41: Possibilidade de realizar e preencher outros questionários disponibilizados pelo IBGE diretamente no aplicativo

    O Requisito Não Implementado, representado pela história de usuário descrita na Tabela XX abaixo, especifica que o aplicativo deve permitir que o usuário visualize, preencha e envie outros questionários disponibilizados pelo IBGE, além do principal, de forma prática e centralizada dentro do próprio sistema. Essa funcionalidade visa ampliar a utilidade do app e facilitar a participação dos usuários em diferentes pesquisas institucionais.

    Tabela 43: US41 – Possibilidade de realizar e preencher outros questionários disponibilizados pelo IBGE diretamente no aplicativo

    ID História de Usuário Critérios de aceitação Rastreabilidade Prioridade
    US41 Eu, como usuário do aplicativo, desejo ter acesso a outros questionários disponibilizados pelo IBGE, para que eu possa preenchê-los diretamente no app de forma prática. - Os questionários adicionais devem estar listados em uma seção específica no app.
    - O preenchimento deve seguir o mesmo padrão de usabilidade do questionário principal.
    - O envio das respostas deve ser confirmado ao final do preenchimento.
    - O sistema deve indicar claramente quais questionários estão disponíveis, pendentes ou finalizados.    		 RF32-A Must
