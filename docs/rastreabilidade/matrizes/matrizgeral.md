# Matriz Geral de Rastreabilidade

## 1. Introdução

<div style="text-align: justify; text-indent: 2cm;">
Uma <b>Matriz de Rastreabilidade</b> é uma técnica usada para documentar infomações de rastreabilidade entre requisitos elicitados e artefatos produzidos ao longo do desenvolvimento do sistema (POHL; RUPP, 2011). Essa matriz é muito vantajosa, pois permite a conexão entre a pré e pós rastreabilidade, facilitando acompanhamento, cobertura e manutenção dos requisitos durante o ciclo de vida do projeto. 
</div>

## 2. Metodologia

<div style="text-align: justify; text-indent: 2cm;">
As matrizes presentes fazem a junção dos esquemas de <a href="https://requisitos-de-software.github.io/2025.1-IBGE/rastreabilidade/matrizes/forwardFrom/">Forward From</a> e <a href="https://requisitos-de-software.github.io/2025.1-IBGE/rastreabilidade/matrizes/backwardfrom/">Backward From</a> para suas respectivas construções. Todos os <a href="https://requisitos-de-software.github.io/2025.1-IBGE/elicitacao/Requisitos/">Requisitos Elicitados</a> são representados na Matriz. 
</div>

## 3. Matriz Geral

### 3.1 Matriz Geral dos Requisitos Funcionais


<div style="text-align: justify; text-indent: 2cm;">
A Matriz Geral dos Requisitos Funcionais conta com 8 colunas, que são:
</div>
<ul style="text-align: justify; padding-left: 4em; margin-top: 0.5em;">
<li> ID - Contendo o ID de cada <b>Requisito</b>
<li> Nome - Contendo o nome/descrição de cada <b>Requisito</b>
<li> Fonte - Liga os <b>Requisitos</b> à sua <b>Técnica de Elicitação</b>, representando a Pré-Rastreabilidade.
<li> UC - Liga o requisito ao <b>artefato de Diagrama de casos de uso</b>, representando a Pós-Rastreabilidade.
<li> C - Liga o requisito ao <b>artefato de Cenários</b>, representando a Pós-Rastreabilidade.
<li> L - Liga o requisito ao <b>artefato de Léxicos</b>, representando a Pós-Rastreabilidade.
<li> US - Liga o requisito ao <b>artefato de História do usuário</b>, representando a Pós-Rastreabilidade.
<li> Status - Se aquele requisito está <b>implementado, não implementado ou não informado(não foi possível fazer a verificação da sua implementação)</b>. 
</ul>

### 3.2 Matriz Geral dos Requisitos Não Funcionais

<div style="text-align: justify; text-indent: 2cm;">
A Matriz Geral dos Requisitos Não Funcionais conta com 6 colunas, que são:
</div>
<ul style="text-align: justify; padding-left: 4em; margin-top: 0.5em;">
<li> ID - Contendo o ID de cada <b>Requisito</b>
<li> Nome - Contendo o nome/descrição de cada <b>Requisito</b>
<li> Fonte - Liga os <b>Requisitos</b> à sua <b>Técnica de Elicitação</b>, representando a Pré-Rastreabilidade.
<li> Especificação Suplementar - Liga o requisito ao <b>artefato de Especificação Suplementar</b>, representando a Pós-Rastreabilidade.
<li> NFR - Liga o requisito ao <b>artefato de NFR Framework</b>, representando a Pós-Rastreabilidade.
<li> Status - Se aquele requisito está <b>implementado, não implementado ou não informado(não foi possível fazer a verificação da sua implementação)</b>. 
</ul>
